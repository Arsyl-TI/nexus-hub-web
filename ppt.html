Latar Belakang Produk

Identifikasi Masalah

Mahasiswa dan profesional sering membutuhkan alat pemroses dokumen (kompresi, merge, konversi). Namun, layanan yang ada seringkali:

  • Mengancam privasi (file disimpan di server tanpa persetujuan).
  • Memiliki batasan ukuran file yang ketat.
  • Terpisah-pisah di berbagai website berbeda.

Solusi: Nexus Hub

Sebuah platform terpusat (*Hub*) yang aman, cepat, dan didesain khusus untuk efisiensi akademis.

Nilai Jual Utama (USP):

Auto Garbage Collection: File langsung dihapus dari memori server setelah diproses.

Data Isolation: Riwayat dokumen diproteksi via Supabase Row Level Security (RLS).

Arsitektur Sistem (Microservice)

Pemisahan layer Client dan Server untuk menjamin skalabilitas dan performa proses komputasi berat (manipulasi PDF).

Strategi Pengujian Perangkat Lunak

Untuk memastikan reliabilitas produk, pengujian dibagi menjadi 3 lapisan (Layers of Testing):

1. Unit Testing (Backend)

Menguji fungsi individual di Python secara terisolasi.

  • Fungsi kalkulasi ukuran file.
  • Validasi MIME type file yang diunggah.

2. Integration Testing

Menguji komunikasi antar microservice.

  • Koneksi Next.js ke Flask API via CORS.
  • Penyisipan log aktivitas ke DB Supabase.

3. System / E2E Testing

Menguji aliran (*flow*) dari sudut pandang pengguna.

  • Proses Login -> Upload File -> Proses -> Download.
