Latar Belakang

Masalah di Lapangan

  • Fragmentasi Tools: Mahasiswa harus membuka web berbeda (seperti iLovePDF) untuk menggabung, mengompres, atau mengubah format dokumen.
  • Risiko Kebocoran Data: Mengunggah dokumen Tugas Akhir (Hak Kekayaan Intelektual) ke server gratisan pihak ketiga sangat berisiko.
  • Kesalahan Format: Skripsi sering ditolak dosen karena hal sepele seperti salah spasi, batas margin, atau font yang luput dari mata.

Proses pengolahan dokumen mahasiswa saat ini sangat tidak efisien, berisiko, dan rawan kesalahan teknis (Human Error).

Rumusan Masalah

  • Bagaimana merancang sebuah platform terpusat yang dapat melakukan manipulasi PDF (Merge, Compress, Ekstrak) dalam satu antarmuka?
  • Bagaimana mengimplementasikan teknologi AI Spasial untuk memvalidasi margin dan tata letak secara otomatis berdasarkan Pedoman Tugas Akhir UIR?
  • Bagaimana merancang arsitektur basis data dan sesi untuk mengisolasi riwayat file agar privasi setiap mahasiswa terjamin mutlak?

Solusi & Fitur Utama

Manipulasi PDF

PDF Merger untuk menggabungkan file, dan Compressor untuk mengecilkan ukuran tanpa merusak kualitas.

Ekstraksi Format

Mengubah kumpulan foto (JPG) menjadi PDF, atau mengubah tata letak PDF menjadi format Microsoft Word.

AI Format Analyzer

Asisten pintar yang memvalidasi margin, spasi, dan font pada makalah berdasarkan pedoman UIR secara instan.

Validator Pedoman UIR (6 Parameter)

AI membaca koordinat spasial (X/Y) teks dalam dokumen untuk memverifikasi aturan baku ini:

1. Margin Kertas (Sembir): Validasi presisi Kiri 4cm, Kanan/Atas/Bawah 3cm.
2. Spasi Baris: Pengecekan jarak antar teks 2 Spasi (Kecuali Abstrak 1 Spasi).
3. Font & Ukuran: Wajib Times New Roman & 12pt (Judul 14pt).
4. Batas Abstrak: Menghitung kata agar tidak melebihi 500 kata.
5. Daftar Isi: Memastikan daftar isi otomatis (bukan ketikan manual).
6. Penomoran Halaman: Transisi Romawi (i, ii) ke angka Arab (1, 2).

Logika "Visual Highlighting"

Cara AI Menemukan Kesalahan:

  • Koordinat X/Y: Python membedah PDF bukan sebagai gambar, melainkan membaca titik koordinat letak tiap huruf (*Bounding Box*).
  • Kalkulasi Jarak: Menghitung jarak Y antar baris (untuk mencari spasi) dan batas sumbu X (untuk mengecek margin 4cm).
  • Draw Rect: Python menggambar kotak warna (merah/oranye) langsung di atas dokumen untuk memvisualisasikan kesalahan.

Antarmuka & Output (Showcase)

Keterangan Visual (UI):

Kotak Merah (Red Box)

Digambar pada teks yang menabrak batas Margin Kiri 4cm atau Font/Ukuran salah.

Kotak Oranye (Orange Box)

Digambar pada paragraf yang terdeteksi spasinya terlalu rapat (Menyimpang dari 2 Spasi).

Arsitektur Microservices

Pemisahan Beban (Decoupling)

  • Frontend (Next.js): Mengelola Antarmuka UI (Client-Side) dan otentikasi login dengan mulus.
  • Backend (Python): Mesin komputasi PyMuPDF menanggung beban berat proses file secara rahasia.

Keamanan Data

  • Data Isolation (Supabase RLS): Pengguna tidak akan pernah bisa mengakses riwayat atau file milik pengguna lain.
  • Garbage Collection: Sistem Python membuang file temporer dari RAM seketika setelah proses (Zero Data Retention).

Terima Kasih

Nexus Hub membuktikan bahwa pemrosesan dokumen kompleks dapat dilakukan dengan efisien, aman, dan cerdas melalui rekayasa perangkat lunak modern.
